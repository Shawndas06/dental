# AI Orchestrator

**Порт:** `8101`  
**Стек:** FastAPI

## Назначение

Маршрутизация текстовых сообщений пациента: классификация intent, подбор врача/услуги, формирование экрана с кнопками слотов. Валидация ответа перед показом пациенту.

## Режимы

| Режим | Переменная | Поведение |
|-------|------------|-----------|
| Rules | `AI_MODE=rules` | CKS на ключевых словах (по умолчанию) |
| OpenAI | `AI_MODE=openai` | LLM + валидатор (нужен `OPENAI_API_KEY`) |

## CKS (Clinical Knowledge System)

Модуль `cks.py` классифицирует текст без вызова LLM:

| Intent | Триггеры | Действие |
|--------|----------|----------|
| `clinical_question` | «диагноз», «болезнь», «чем лечить» | Безопасный отказ |
| `unknown_service` | «имплант», «брекеты», «отбеливание» | Услуги нет в клинике |
| `appointment_intake` | боль, гигиена, удаление и т.д. | Маршрут к врачу и слотам |
| `low_confidence` | прочее | Уточняющий вопрос или меню |

Маршрутизация возвращает `doctor_id`, `service_id`, `visit_type`, `confidence`.

## Эндпоинт

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/ai/intake` | Текст → `IntakeResponse` (Screen + route) |

### Пример запроса

```json
{
  "telegram_user_id": "1001",
  "text": "Болит зуб слева уже неделю"
}
```

### Пример ответа

```json
{
  "text": "Похоже на первичный приём к терапевту...",
  "intent": "appointment_intake",
  "buttons": [
    {"text": "2026-07-01 10:00", "callback_data": "slot:select:slot_doc_therapist_1_0"}
  ],
  "route": {
    "doctorId": "doc_therapist",
    "serviceId": "svc_primary",
    "visitType": "primary_consultation"
  }
}
```

## Safety Validator

Модуль `safety.py` проверяет ответ перед отправкой:

- нет диагнозов и назначений лечения;
- кнопки слотов ссылаются на реальные `slot_id` из Core API;
- нет выдуманных услуг и цен.

При провале валидации публикуется событие `ai.output.validation_failed`.
