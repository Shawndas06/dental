# Сценарии пользователя

## Регистрация

При первом `/start` пациент видит приветствие и кнопку **Продолжить**.

| Шаг | Действие | Результат |
|-----|----------|-----------|
| 1 | `/start` | Экран приветствия |
| 2 | **Продолжить** | `POST /api/patients/quick-register` — имя из профиля Telegram |
| 3 | — | Экран «Регистрация завершена» + кнопка **Записаться** |

Цены и контакты доступны **без регистрации**. Запись и текстовый intake — только после регистрации.

Опционально: пациент может отправить контакт (кнопка «Поделиться номером» в Telegram) — телефон сохранится в профиле.

## Главное меню

```
Демо стоматология

Выберите раздел:
[ Записаться ] [ Мои записи ]
[    Цены    ] [  Контакты  ]
[      Календарь врача       ]
```

| Кнопка | Сценарий |
|--------|----------|
| Записаться | Выбор услуги → слоты → подтверждение |
| Мои записи | Список активных записей, перенос, отмена |
| Цены | Прайс-лист из Core API |
| Контакты | Телефон клиники |
| Календарь врача | Расписание демо-врача |

## Запись через кнопки

1. **Записаться** → список услуг клиники.
2. Выбор услуги → до 6 ближайших свободных слотов.
3. Нажатие слота → создание записи (только кнопкой, не текстом).
4. При занятом слоте — сообщение об ошибке и предложение выбрать другой.

## Запись через текст (гибридный intake)

Пример: `Болит зуб слева уже неделю`

1. Текст уходит в **ai-orchestrator**.
2. CKS определяет intent: `appointment_intake`.
3. Подбирается врач, услуга, тип визита.
4. Бот показывает **реальные слоты** кнопками.
5. Пациент нажимает слот — запись создаётся в Core API.

!!! info "Граница AI"
    AI не создаёт запись. Он только маршрутизирует и показывает кнопки.

## Мои записи

Для каждой активной записи:

| Кнопка | Действие |
|--------|----------|
| Перенести · HH:MM | Выбор нового слота |
| Отменить · HH:MM | Отмена записи |

Внизу экрана: **Назад** и **Главное меню**.

## Перенос врачом (approval flow)

1. Врач инициирует перенос через API (`reschedule/request-by-doctor`).
2. Пациент получает кнопки **Одобрить** / **Отклонить**.
3. При одобрении — запись обновляется; при отклонении — сохраняется текущая.

## Граничные сценарии

### Клинический вопрос

**Ввод:** `Что у меня за болезнь?`

**Ответ:** безопасный отказ без диагноза, предложение записаться на очный приём.

### Неизвестная услуга

**Ввод:** `Хочу имплантацию`

**Ответ:** услуга отсутствует в профиле клиники — без выдуманных цен и слотов.

### Занятый слот

Два пациента нажимают один слот — второй получает `409 Conflict` и сообщение о занятости.

## Демо-сценарии для проверки

| # | Сценарий | Ожидание |
|---|----------|----------|
| 1 | Текст «Болит зуб слева уже неделю» | Слоты терапевта, кнопки `slot:select:...` |
| 2 | Нажатие слота | Запись создана, событие в NATS |
| 3 | «Что у меня за болезнь?» | Отказ без диагноза |
| 4 | «Хочу имплантацию» | Нет галлюцинаций об услуге |
| 5 | Мои записи → Перенести | Кнопочный перенос без AI |

## Уведомления (worker)

| Событие | Получатель |
|---------|------------|
| `appointments.created` | Пациент, staff-группа |
| `appointments.cancelled` | Пациент, staff |
| `appointments.rescheduled.*` | Пациент, врач |
| `reminders.scan` | Напоминание за 24 часа |

Для staff-уведомлений задайте `STAFF_GROUP_CHAT_ID` в `.env`.
