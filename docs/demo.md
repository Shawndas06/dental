# Демо-сценарий

Автоматическая проверка ключевых возможностей бота для презентации заказчику или приёмки MVP.

## Зачем нужен

Демо прогоняет полный цикл без ручных действий в Telegram:

- регистрация пациента
- AI-маршрутизация по жалобе
- запись через кнопку слота
- граничные случаи (диагноз, неизвестная услуга)
- экран «Мои записи»

## Способы запуска

### 1. Админ-панель (рекомендуется)

1. Откройте `http://127.0.0.1:8190/demo`
2. Нажмите **Запустить демо**
3. Просмотрите таблицу шагов и JSON-результат

Кнопка **Только сброс** — очистка данных без прогона сценария.

### 2. CLI

```bash
python3 -m scripts.run_demo_scenario
```

Опции:

```bash
python3 -m scripts.run_demo_scenario --no-reset   # без предварительного сброса
python3 -m scripts.run_demo_scenario --core-api http://127.0.0.1:8100
```

Код выхода: `0` — все шаги успешны, `1` — есть ошибки.

### 3. Admin API

```bash
curl -X POST http://127.0.0.1:8100/api/admin/demo/run \
  -H "X-Admin-Token: your-token"
```

## Предварительные условия

```bash
docker compose up -d core-api bot-gateway ai-orchestrator admin
```

| Сервис | Порт | Обязателен |
|--------|------|------------|
| core-api | 8100 | да |
| bot-gateway | 8180 | да |
| ai-orchestrator | 8101 | да |
| admin | 8190 | для UI |

## Шаги сценария

| # | Шаг | Что проверяет |
|---|-----|---------------|
| 1 | Сброс демо-данных | Чистая база, слоты `available` |
| 2 | Быстрая регистрация | `POST /api/patients/quick-register` |
| 3 | AI intake | «Болит зуб слева уже неделю» → intent + слоты |
| 4 | Бот `/start` | Главное меню для зарегистрированного |
| 5 | Запись через слот | `slot:select:` → запись в Core API |
| 6 | Отказ от диагноза | «Что у меня за болезнь?» |
| 7 | Неизвестная услуга | «Хочу имплантацию» |
| 8 | Мои записи | Список активных записей в боте |

Демо-пользователь: Telegram ID **`999000001`**.

## Ручная проверка в Telegram

После автоматического демо можно повторить вручную:

1. `/start` → **Продолжить**
2. Напишите: `Болит зуб слева уже неделю`
3. Нажмите слот → запись создана
4. **Мои записи** → перенос / отмена

Или кнопочный путь: **Записаться** → услуга → слот.

## Пример успешного результата

```json
{
  "success": true,
  "telegramUserId": "999000001",
  "steps": [
    {"name": "Быстрая регистрация", "ok": true, "detail": "Пациент Демо Клиент создан"},
    {"name": "AI intake по жалобе", "ok": true, "detail": "Intent: appointment_intake, слотов: 4"},
    {"name": "Запись через кнопку слота", "ok": true, "detail": "Запись подтверждена..."}
  ]
}
```

## Что делать при ошибках

| Симптом | Решение |
|---------|---------|
| Шаг «AI intake» FAIL | Проверьте `ai-orchestrator` на `:8101` |
| Шаг «Запись через слот» FAIL | Нет свободных слотов → **Только сброс** в админке |
| 401 на admin API | Неверный `ADMIN_TOKEN` в `.env` и cookie |
| Connection refused | `docker compose ps`, поднимите сервисы |
| Бот не отвечает в Telegram | Отдельно: `TELEGRAM_BOT_TOKEN`, polling, прокси |
